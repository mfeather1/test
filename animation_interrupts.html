Animation Interrupts

When the stop button is pressed while a sequence is playing the animation will stop immediately as as can be seen on the first cube below by pressing play and then stop (middle button). On the second cube a bit of javascript is used to handle the interrupt so that animation of the current move will complete instead of stopping immediately. [note: the second sequence is to show that there is no reset bug, that will be removed prior to installing this on the AnimCubeJS repo].



On the first cube below, each subsequent input move (via buttons or letters) will interrupt the currently playing move if entered quickly as can be seen on the first cube below. With a bit of javascript, input moves do not interrupt animation already in progress (they queue up) as can be seen on the second cube.
